## Data Preprocessing {#Data-Preprocessing}

Real-world data is rarely clean. Before handing anything to a model, we need to handle missing values, normalize scales, and convert categorical variables into numbers. Skipping any of these steps is one of the most common reasons models underperform in production.

### Imputation {#Imputation}

Instead of dropping rows with missing values and losing information, we replace them with a computed substitute using `SimpleImputer`.
```python
# strategy options: mean | median | most_frequent | constant
from sklearn.impute import SimpleImputer

imputer = SimpleImputer(strategy='median')
X_imp   = imputer.fit_transform(X_train)

# For categorical columns
cat_imp = SimpleImputer(strategy='most_frequent')
X_cat   = cat_imp.fit_transform(X_cat_train)
```

### Feature Scaling {#Feature-Scaling}

Normalizing feature ranges ensures no variable dominates due to its magnitude. This is critical for distance-based models like KNN and SVM, and for gradient descent algorithms.
```python
from sklearn.preprocessing import StandardScaler, MinMaxScaler

# StandardScaler → mean=0, std=1
scaler    = StandardScaler()
X_sc      = scaler.fit_transform(X_train)
X_test_sc = scaler.transform(X_test)   # never refit on test data!

# MinMaxScaler → range [0, 1]
mm   = MinMaxScaler()
X_mm = mm.fit_transform(X_train)
```

Use `StandardScaler` for SVM, KNN, and neural networks. Use `MinMaxScaler` when you need a bounded range. Tree-based models like RandomForest require no scaling at all.

### Encoding {#Encoding}

Machine learning models require numeric input. We convert categorical labels using either `LabelEncoder` for ordinal data or `OneHotEncoder` for nominal categories.
```python
from sklearn.preprocessing import LabelEncoder, OneHotEncoder

# Label Encoding (ordinal: Low → 0, Medium → 1, High → 2)
le = LabelEncoder()
y  = le.fit_transform(df['risk_level'])

# One-Hot Encoding (nominal: no false ordinal relationship)
ohe   = OneHotEncoder(sparse_output=False, drop='first')
X_cat = ohe.fit_transform(df[['city']])
```

> **⚠️ Data Leakage Warning:** Always `fit` your scaler and imputer *only* on training data, then `transform` both train and test sets. Fitting on test data leaks information and produces inflated, misleading evaluation scores.

## K-Fold Cross-Validation {#K-Fold-Cross-Validation}

A single train/test split is fragile — your score depends heavily on which samples happened to land in which set. K-Fold solves this by rotating the validation window across K partitions, giving you a mean score and standard deviation you can actually trust.

How it works with K=5:

- **Fold 1:** Validate on partition 1, train on 2–5
- **Fold 2:** Validate on partition 2, train on 1, 3–5
- **Fold 3:** Validate on partition 3, train on 1–2, 4–5
- **Fold 4:** Validate on partition 4, train on 1–3, 5
- **Fold 5:** Validate on partition 5, train on 1–4

The final reported score is the **mean of all 5 validation scores**. The standard deviation tells you how stable your model is across different data slices.
```python
from sklearn.model_selection import cross_val_score, StratifiedKFold, cross_validate
from sklearn.ensemble import RandomForestClassifier

model = RandomForestClassifier(n_estimators=100, random_state=42)

# Basic 5-fold CV
scores = cross_val_score(model, X, y, cv=5, scoring='accuracy')
print(f"Mean: {scores.mean():.4f} ± {scores.std():.4f}")

# Stratified K-Fold — preserves class distribution in each fold
skf    = StratifiedKFold(n_splits=5, shuffle=True, random_state=42)
scores = cross_val_score(model, X, y, cv=skf, scoring='f1')

# Multiple metrics in one call
res = cross_validate(model, X, y, cv=5,
    scoring=['accuracy', 'precision', 'recall', 'f1', 'roc_auc'])
print(f"F1:      {res['test_f1'].mean():.4f}")
print(f"ROC-AUC: {res['test_roc_auc'].mean():.4f}")
```

Use `K=5` for large datasets and `K=10` for smaller ones. Always use `StratifiedKFold` when your target classes are imbalanced — it ensures each fold has the same class ratio as the full dataset.

## Hyperparameter Tuning with GridSearchCV {#Hyperparameter-Tuning}

Hyperparameters are settings defined *before* training — like the number of trees in a forest or the regularisation strength of an SVM. `GridSearchCV` tries every combination in a defined grid, evaluates each with cross-validation, and returns the best configuration automatically.
```python
from sklearn.model_selection import GridSearchCV
from sklearn.svm import SVC

param_grid = {
    'C':      [0.1, 1, 10, 100],
    'kernel': ['linear', 'rbf', 'poly'],
    'gamma':  ['scale', 'auto', 0.01, 0.001]
}
# 4 × 3 × 4 = 48 combinations × cv=5 = 240 model fits

grid = GridSearchCV(
    estimator  = SVC(),
    param_grid = param_grid,
    cv         = 5,
    scoring    = 'accuracy',
    n_jobs     = -1,
    refit      = True
)
grid.fit(X_train, y_train)

print(grid.best_params_)
print(f"Best CV score: {grid.best_score_:.4f}")
test_acc = grid.best_estimator_.score(X_test, y_test)
```

For large search spaces, `RandomizedSearchCV` samples `n_iter` random combinations instead of all of them.
```python
from sklearn.model_selection import RandomizedSearchCV

rand = RandomizedSearchCV(SVC(), param_grid, n_iter=20, cv=5, n_jobs=-1)
rand.fit(X_train, y_train)
```

## Building ML Pipelines {#Building-ML-Pipelines}

A `Pipeline` chains all preprocessing steps and the model into a single object. It prevents data leakage during cross-validation, makes `GridSearch` work on preprocessing parameters, and means deployment is as simple as calling `.predict()` on raw data.

The flow: **Raw Data → Imputer → Scaler → Encoder → Model → Prediction**
```python
from sklearn.pipeline import Pipeline
from sklearn.compose import ColumnTransformer
from sklearn.impute import SimpleImputer
from sklearn.preprocessing import StandardScaler, MinMaxScaler, OneHotEncoder
from sklearn.ensemble import RandomForestClassifier
from sklearn.model_selection import GridSearchCV

num_features = ['age', 'income', 'credit_score']
cat_features  = ['employment_type', 'region']

num_transformer = Pipeline(steps=[
    ('imputer', SimpleImputer(strategy='median')),
    ('scaler',  StandardScaler())
])

cat_transformer = Pipeline(steps=[
    ('imputer', SimpleImputer(strategy='most_frequent')),
    ('encoder', OneHotEncoder(handle_unknown='ignore', sparse_output=False))
])

preprocessor = ColumnTransformer(transformers=[
    ('num', num_transformer, num_features),
    ('cat', cat_transformer, cat_features)
])

pipeline = Pipeline(steps=[
    ('preprocessor', preprocessor),
    ('classifier',   RandomForestClassifier(random_state=42))
])

param_grid = {
    'classifier__n_estimators':  [100, 200, 300],
    'classifier__max_depth':     [None, 10, 20],
    'preprocessor__num__scaler': [StandardScaler(), MinMaxScaler()]
}
grid = GridSearchCV(pipeline, param_grid, cv=5, scoring='f1', n_jobs=-1)
grid.fit(X_train, y_train)
y_pred = grid.best_estimator_.predict(X_test)
```

## Saving & Deploying with .pkl {#Saving-and-Deploying}

Saving a *Pipeline* object preserves all preprocessing steps. Raw data can be predicted on directly with no manual transformation in production.
```python
import joblib

joblib.dump(pipeline, 'fraud_detector_v1.pkl')
```
```python
import joblib
import pandas as pd

pipeline = joblib.load('fraud_detector_v1.pkl')

new_data = pd.DataFrame([{
    'age': 34,
    'income': None,
    'credit_score': 720,
    'employment_type': 'salaried',
    'region': 'north'
}])

pred  = pipeline.predict(new_data)
proba = pipeline.predict_proba(new_data)[0, 1]
print(f"Prediction: {pred[0]} | Confidence: {proba:.2%}")
```
```python
from flask import Flask, request, jsonify
import joblib, pandas as pd

app      = Flask(__name__)
pipeline = joblib.load('fraud_detector_v1.pkl')

@app.route('/predict', methods=['POST'])
def predict():
    data  = request.get_json()
    df    = pd.DataFrame([data])
    pred  = pipeline.predict(df)[0]
    proba = pipeline.predict_proba(df)[0, 1]
    return jsonify({
        'prediction': int(pred),
        'confidence': round(float(proba), 4),
        'model_ver':  'v1.0.0'
    })

if __name__ == '__main__':
    app.run(host='0.0.0.0', port=5000)
```
